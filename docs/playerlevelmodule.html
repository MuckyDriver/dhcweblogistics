Getting Started


Install

To get this module just click here to get the download file (.rbxm)
Next, you can copy this .rbxm file into a roblox place which you are working on. Furthermore, move it to "ReplicatedStorage".

About

This module was created to make a player levelling system a really easy thing to do. Using this module you can update and change how the player levels up.



Properties


Level

Level is the current level number assigned to the player.

XP

XP is the current XP amount assigned to the player.

Multi

Mulii/Multiplier property is a number which is used for the multiplication factor of the assigned player levelling up speed. The bigger the multiplier the harder each level is. Multi can be taken the same as starting point. E.g. 100 Multi = 100 XP to complete Level 1, it will increase from there.

Client

Read OnlyClient property is the assigned Player Object.



Functions


new

Function {.new} will create a new class for the player. This function will create your player level assigned to the player.

local Levels = require(Module)

Players.PlayerAdded:Connect(function(Player: Player)
local PlayerLevel = Levels.new(Player)
end)

Update

Function {:Update} is to add a new amount of experience to the player. Example below adds 1000 xp to the players level!

PlayerLevel:Update(1000)

AtMaxLevel

Function {:AtMaxLevel} is to check if the current level is the max level. If so it will stop the player from levelling up any further.

if PlayerLevel:AtMaxLevel() then
print("Reached max level!")
end

SetMaxLevel

Function {:SetMaxLevel} to set a custom max level. Max level means that the {:AtMaxLevel} function will be able to check against it to see if you have reached max level yet.

PlayerLevel:SetMaxLevel(100)

OverrideCurrentLevel

To override your current level you can use the function {:OverrideCurrentLevel}. However, do not use this if you want the XP to be like it was before. For any stored data please use {:OverrideXPAmount} since it will update the players level from what the xp amount is.

PlayerLevel:OverrideCurrentLevel(15)

OverrideXPAmount

To override the current xp amount you can use the function {:OverrideXPAmount}. Unlike {:OverrideCurrentLevel} this will change the xp amount. Also, using this function will return back to original place in the progress of the level the player is currently in.

PlayerLevel:OverrideXPAmount(25000)

GetXPNeededForNextLevel

This function will return the needed xp you need to level up to the next level. This function is mainly used in the backend part of the module, this is used when you update the player xp amount.

CreateLeaderboard

{:CreateLeaderboard} function will create leaderstats using the player level values. You can choose to have it dynamic or not - dynamic will update automatically.

local Auto: boolean = true
local Interval: number = 1

PlayerLevel:CreateLeaderboard(Auto, Interval)

ProgressInLevel

This function would return a decimal percentage of how far the player is through their current level. This function can be used to create progress bars on using a local script.

local Progress: number = Levels.ProgressInLevel(PlayerLevel)

GetPlayerLevel

{:GetPlayerLevel} will return the passed player created player level data. This can only work if the player has already been created.

Players.PlayerRemoving:Connect(function(Player: Player)
local PlayerLevel = Levels.GetPlayerLevel(Player)
print("Finished on level:", PlayerLevel.Level)
end)
